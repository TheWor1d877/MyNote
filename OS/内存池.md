内存池（Memory Pool）是预先分配一大块内存，然后自行管理分配和释放的技术，用于解决标准内存分配器（如malloc/free）的性能问题。

## 核心思想
- 用空间换时间，用确定性换随机性”
一次性向系统申请大块内存
自己切成小块分配给应用
释放时不真正还给系统，而是回收到池中复用

“内存池通过预先分配大块内存并自行管理，解决了malloc的四个问题：1）系统调用开销；2）内存碎片；3）线程锁竞争；4）分配不确定性。特别适合高频小对象分配场景，如游戏粒子、网络连接池等，性能可提升10倍以上。”

## 现代替代方案
1. tcmalloc / jemalloc
```cpp
// 已经内置了高效的内存池机制
// 直接链接即可享受优化
// -ltcmalloc 或 -ljemalloc
```

2. C++17 内存资源
```cpp

#include <memory_resource>
std::pmr::synchronized_pool_resource pool;  // 标准库自带池
std::pmr::vector<int> vec(&pool);           // 容器使用指定内存池
```
